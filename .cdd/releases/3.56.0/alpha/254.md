## Design decision worth recording (not a finding)

**Lookup order chose noun-verb first, flat fallback second.** The alternative (flat first) would have kept existing flat behavior bit-identical as the dominant path. Noun-verb-first means `cn kata run` is the primary surface; `cn kata-run` is resolved only when the noun-verb lookup misses.

This matters when a `<noun>` is itself a registered flat command AND a `<noun>-<verb>` is also registered. The current kernel has no such conflict (`deps` is flat; no `deps-lock` registered). If a future `deps-lock` were added while `deps` remained, `cn deps lock` would prefer `deps-lock`. That matches the git convention (`git remote add` routes to `remote-add`, not `remote` with arg `add`).

Documented in `dispatch.go:40ā€“52` ("Lookup order") so the precedence is discoverable. Ī² confirmed `DepsCmd` internal dispatch is preserved via `TestResolveCommandDepsSubcommandPreserved`. No action needed.
