**P2 вЂ” Render-capture seam Option A makes the bytes-equal property structural, not merely tested.** When the operator-visible invariant is "default path output unchanged", two implementation shapes satisfy the test oracle but only one satisfies the property by construction: (a) capture into a buffer always, write the buffer to stdout in the default arm вЂ” the bytes-equal property holds *if and only if* the buffer-to-stdout copy is byte-faithful (a test oracle catches divergence, but the shape admits divergence); (b) pass `inv.Stdout` straight through in the default arm, buffer only in the spawn arm вЂ” the default arm is bytes-identical to the prior-version default arm by construction (the buffer is not on the default code path). Оі's pre-flagged failure mode #6 (silent renderer change) is inoculated by Option A's structural shape, not by a test. The pattern generalizes: when a "preserve X exactly" invariant has a "minimum surface change" implementation and a "uniform surface change" implementation, the minimum-surface implementation moves the invariant from oracle-checked to structurally-preserved.

**P3 вЂ” Pre-render-ordering proven by absence-of-render-diagnostic, not by code reading.** The "nonexistent HUB_DIR" oracle shape (`TestActivate_MutualExclusion_FiresBeforeRender`, `TestActivate_MissingBinary_FiresBeforeRender`) proves pre-render ordering by what stderr does *not* contain.
