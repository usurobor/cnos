## The I6 classification-transition finding and its resolution

Finding (see `beta-review.md` §R0 Findings 1): α's self-coherence.md AC5 evidence claimed "byte-identical" `cn cdd verify --unreleased` output between `origin/main` and `cycle/569`, but that comparison ran the check *without* `--exceptions` — the actual CI invocation (per `.github/workflows/build.yml`'s `cdd-artifact-check` job) uses `--exceptions .cdd/exceptions.yml` and, run that way, `cycle/569` showed one additional failure over `origin/main`: `.cdd/unreleased/569/self-coherence.md` itself failing the section-header check.

Traced the root cause to `classifyCycleType()` (`ledger.go` line 424): a cycle classifies as "small-change" (hard-fail section validation) when `self-coherence.md` exists but `beta-review.md` does not yet — exactly cycle/569's state at the moment I started the review, before this file existed on the branch. Confirmed this is not an α regression by diffing #569's and #570's `self-coherence.md` header conventions (identical `## §`-prefixed headings) against their differing classification outcomes (#570 already has `beta-review.md` and gets the lenient "triadic" ⚠️-only path; #569 didn't yet). Re-ran the check after writing `beta-review.md` (which flips #569's own classification) and confirmed parity restored. **Resolution:** not treated as a fix-round-blocking finding — it self-resolves the moment `beta-review.md` lands in the same commit set the actual gating CI run will see, which is guaranteed by the time δ opens the PR. Recorded as an investigated, non-blocking finding rather than silently absorbed, and flagged to γ as a recurring tooling gap worth a follow-up issue (see `gamma-closeout.md` triage) rather than an in-cycle fix, since the fix surface (`cdd-verify`'s classification/section-detection logic) is outside #569's pinned package scope.
