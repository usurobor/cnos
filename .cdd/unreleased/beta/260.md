### Finding 1 — F4 sibling drift in gamma/SKILL.md (C, judgment)

**Caught:** Round 1 via review §2.2.1 sibling-enumeration and §2.2.8 authority-surface conflict.

**Surface:** α's PR body scoped F4 (`Active invariants` → `Active design constraints`) to the issue template + issue/SKILL.md — the two files whose field definition was being renamed. But three sibling references in gamma/SKILL.md (L144, L159, L381) still carried the old term, and L144 explicitly promised "γ does name Tier 3 skills, active invariants, related artifacts … **exactly as `issue/SKILL.md` requires**" — a direct authority conflict with the renamed field. An α following gamma literally would have omitted transition/process constraints that the renamed template field was designed to capture.

**Fix:** α pushed `456e164` with three exact-string replacements, scope-clean (6 line-diff, no scope creep). Verified post-fix: `git grep 'active invariants' gamma/SKILL.md` → zero hits. The one remaining `Active invariants` occurrence in review/SKILL.md L289 is correct contextual usage (naming one of three sub-categories of design constraints: invariants / transition / process) and was explicitly preserved by α with justification. Clean narrow Round-2 approval.

**Root cause:** rename cycles that touch a named field should enumerate every skill that references the field by name, not only the skills in whose section the field is defined. The F4 bullet in the PR body said "issue template + issue skill" as though that was the full surface — it was not. The review skill §2.2.1 sibling-enumeration rule would have caught this at authoring time if α had run it on the renamed term.

**Severity:** C (real incoherence, non-blocking) because a literal reader of gamma/SKILL.md would ship narrower-than-intended issue packs, but the runtime effect is a skill-doc mismatch, not a code path that breaks. Severity convention matches F2/F3 in this same PR (also cross-file drift fixes rated C).

**Triage disposition:** **drop** as a one-off; the sibling-enumeration rule that would have prevented it (`review §2.2.1`) is already in the review skill as written. This cycle demonstrates the rule caught the gap at review time — the skill worked. Not an MCA candidate.
