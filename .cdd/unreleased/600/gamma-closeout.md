## Cycle iteration

**Root cause.** `alpha/SKILL.md` §2.3's peer-enumeration rule enumerates several concrete peer classes but did not, before this cycle, name CI-workflow-file comments as a distinct peer class alongside script-header comments and doctrine prose. When a mechanism's description lives in three places — script header, `*/SKILL.md` doctrine, and a live workflow file's inline comment/step name — a grep scoped to "doc/skill surfaces" structurally cannot see the third. The rule's own derivation history (#266, #283) already establishes the general pattern ("one fact, several sibling surfaces, fix some not all"); this cycle is a second, narrower instance of the same underlying gap, now specifically at the doc-surface / CI-workflow-file boundary rather than the role-skill / lifecycle-skill boundary #283 named.

**Disposition: patch landed now.** γ added a targeted bullet to `alpha/SKILL.md` §2.3's "Mandatory cases" list — "CI-workflow-file comments" — naming `.github/workflows/*.yml` job comments and step names as an explicit peer class of script-header comments and doctrine prose, with a `Derives from: #600 β R1 F1` citation matching the existing bullet style (see the adjacent "skill-class peers" bullet for the precedent this follows). This is a small, targeted addition to the existing bullet list — not a rewrite of §2.3 — per gamma/SKILL.md §3.6 ("land immediate process fixes in the same cycle when possible") and the correction is clear enough not to need a separate cycle: the recurrence is now named twice (#283, #600) at two different radii, which is exactly the bar CDD's own escalation logic uses elsewhere (a second occurrence of the same class converts a one-off into a pattern worth codifying). No concrete next MCA is committed beyond this landed patch — the gap is closed at the doctrine level; whether it holds on the next cycle that touches a CI-workflow-file peer is the natural verification point, not a new open action.

**Was a discretionary two-pronged option considered?** Yes: γ could instead have deferred this to a future MCA (a filed issue: "broaden §2.3 peer-enumeration examples"). Rejected in favor of landing now because the fix is genuinely small (one bullet, matching existing structure and citation style), the evidence for the gap is fully in hand from this cycle's own close-outs (no further investigation needed), and deferring a one-bullet doc fix to a future cycle when the finding, root cause, and exact wording are already settled would itself be the "closure amnesia" failure mode gamma/SKILL.md §1 warns against.
