# Issue board maps

Interactive, self-contained visualizations of the `usurobor/cnos` open-issue
board, generated by **`cn issues map`** (Go) and labeled per
[the issue taxonomy](../issues/TAXONOMY.md).

> **Generated file — do not edit by hand.** Regenerate with
> `cn issues map --repo usurobor/cnos --out docs/development/board`, or let the
> `board-map` GitHub Action refresh it on issue changes.

[`index.html`](index.html) is a **PivotViewer-style faceted issue browser** —
one self-contained page (D3 inlined) with three views:

- **Board** — a weighted **Voronoi tessellation** where **cell area = effort**
  and **color = priority**; group-by pivot (kind → area → issue, or reorder by
  priority / status / area / effort), click a group to semantic-zoom in via the
  breadcrumb, click a cell for the issue detail panel.
- **Heatmap** — a `kind × area` matrix of **summed effort**; click a cell to
  filter the whole board.
- **List** — the filtered issues as a table.

A left **facet rail** (kind / area / priority / effort / status / protocol /
dispatchable / unestimated / assignee / age) filters every view at once, with
live counts; search and Export JSON are in the footer.

`board-data.json` is the same records as a standalone file. Open `index.html`
directly — no server needed. Live preview: https://raw.githack.com/usurobor/cnos/main/docs/development/board/index.html

## Encoding

- **Cell area = effort weight** (`effort/S|M|L|XL` = 1/2/4/8).
- **Color = priority** (P0 → P3; `none` neutral).
- **Data is effort-aware**: every record carries `effort`, `effort_weight`, and
  `unestimated`. Unestimated issues render **hatched/dimmed** with nominal weight
  1 (a visible gap, never treated as small) and are selectable via the
  **unestimated only** facet.
- **`kind/tracking`** issues are containers and are excluded from the effort
  rollups (the `effort Σ` stat and the heatmap sums).

## Source of truth

The generator is the Go command `cn issues map`
(`src/packages/cnos.issues/commands/issues-map/`).
There is no standalone script generator on the production path; the browser only
renders the embedded data.
