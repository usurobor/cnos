# Issue board maps

Interactive, self-contained visualizations of the `usurobor/cnos` open-issue
board, generated by **`cn issues map`** (Go) and labeled per
[the issue taxonomy](../issues/TAXONOMY.md).

> **Generated file — do not edit by hand.** Regenerate with
> `cn issues map --repo usurobor/cnos --out docs/development/board`, or let the
> `board-map` GitHub Action refresh it on issue changes.

- [`index.html`](index.html) — **Voronoi tessellation**: organic cells for
  `kind → area → issue`, colored by priority, with facet filters (kind / area /
  priority / status / dispatchable / unestimated / search). Click a kind label to
  zoom, a cell to open the issue; light/dark.
- [`pivot.html`](pivot.html) — **Pivot-feel deep-zoom**: a continuous card wall,
  whole board → single issue card, driven by `d3.zoom()`; the same facet filters
  rearrange the wall.
- `board-data.json` — the embedded board data as a standalone file (the same
  records spliced into the HTML).

Each HTML file is fully self-contained (D3 inlined) — open it directly, no server
needed. For a live preview URL without hosting setup:

- Voronoi: https://raw.githack.com/usurobor/cnos/main/docs/development/board/index.html
- Pivot: https://raw.githack.com/usurobor/cnos/main/docs/development/board/pivot.html

## Encoding

- **Color = priority** (P0 → P3; `none` neutral).
- **Cell area = issue count** in v1; the effort-weighted area encoding
  (`area = effort`, per the `effort/*` labels) is tracked in the follow-up.
- **Data is effort-aware now**: every record carries `effort`, `effort_weight`,
  and `unestimated`. Unestimated issues render with a **dashed cell** (Voronoi)
  / an `unestimated` tag (Pivot) and are selectable via the **unestimated only**
  filter — a visible gap, never treated as small.
- **`kind/tracking`** issues are containers and are excluded from the effort
  rollup (the `effort Σ` stat).

## Source of truth

The generator is the Go command `cn issues map` (`src/go/internal/issuesmap/`).
There is no standalone script generator on the production path; the browser only
renders the embedded data.
