# 3.62.0 — γ creates the cycle branch

## Outcome

Coherence delta: C_Σ A- (`α A-`, `β A-`, `γ B+`) · **Level:** `L7 (diff: L7; cycle cap: L6)`

The cycle-coordination surface is now a single canonical named branch. Pre-#287, α opened a branch under the harness-encoded `claude/{slug}-{rand}` shape, and γ + β had to glob-match `'origin/claude/*'` to discover it — a glob that silently failed when the issue number was not in the branch name (#274 / #283 R1 F1). Post-#287, γ creates `cycle/{N}` from `origin/main` *before* dispatch, the dispatch prompt names `Branch: cycle/{N}` explicitly, and α + β only check out, never create. Polling targets `origin/cycle/{N}` directly with no glob; the branch-discovery friction class is eliminated for all future cycles.

## Why it matters

Three cycles paid for this fix. #274's β + γ both armed `*230*`-shaped globs that matched zero branches because the harness branch was `claude/alpha-tier-3-skills-IZOsO`. #283's β received a separate harness-given β-side branch with α-style "develop and commit" instructions, refused per `beta/SKILL.md` §1, and then had to cherry-pick its R1 verdict onto α's cycle branch. #287 closes the source: γ owns branch creation under the canonical `cycle/{N}` name; the dispatch prompt names the branch; α + β never invent or accept any other name. One cycle = one branch = one named target for all polling.

The cycle implementing this change exemplifies its own rule: γ created `origin/cycle/287` before α was dispatched; α `git switch`ed onto it from the harness pre-provisioned `claude/alpha-cdd-skill-1aZnw`; β switched from `claude/cnos-skill-module-x9jTE` and committed all verdict commits on `cycle/287`. The contract is enforceable by inspection — every commit reachable from the merge has its expected role identity (`alpha@cdd.cnos` / `beta@cdd.cnos` / `gamma@cdd.cnos`) on the canonical branch.
