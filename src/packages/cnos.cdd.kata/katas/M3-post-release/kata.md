# M3 — Post-release Kata

**Class:** method
**Default level target:** L6 / L7
**Purpose:** Prove release closure and post-release assessment quality.

## Scenario

Given a completed release, produce a post-release assessment under two modes (baseline / CDD) and compare closure quality, measurement rigor, and learning capture.

## Required artifacts

- Coherence measurement (baseline, this release, delta)
- Encoding lag table (all open design/process issues)
- MCI/MCA balance decision
- Process learning (what went wrong, what went right, skill patches)
- Review quality metrics
- CDD self-coherence scoring
- Production verification scenario
- CDD closeout trace
- Next move commitment
- Hub memory (daily reflection + adhoc thread)

## Scoring

- L5: assessment exists, scores assigned
- L6: lag table complete, MCI/MCA decision justified, production verification concrete, all immediate outputs executed
- L7: skill patches shipped from findings, level evidence derived from rubric not vibes, closure evidence complete with links
