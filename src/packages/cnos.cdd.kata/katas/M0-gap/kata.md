# M0 — Gap

**Class:** method

Prove that the system can frame the right incoherence before design starts.

## Scenario

Given a bounded change request and surrounding repo state, produce a gap statement under two modes:

- baseline
- cdd

## Required artifacts

- selected gap
- named incoherence
- why it matters
- what fails if skipped
- mode selection
- active skills
- if using CDD mode: CDD trace through step 5

## Scoring

Scored by the judge against rubric.json. Level derivation:

- L5: local problem correctly named
- L6: gap selection reflects surrounding system state and constraints
- L7: selected gap correctly identifies a boundary-level incoherence that changes future work shape
